## 1. 产品概述

「拼车剧本杀」是一款面向同城剧本杀包场局头和车主玩家的社交化车源招募移动 App，解决多人包场活动中"谁有车、怎么坐、怎么走"的组织痛点。局头发布活动时可同步招募车源，车主一键响应，系统智能分组生成去程/返程车队名单，并支持集合打卡与临时变更实时通知。

- 目标用户：大学生社团、熟人剧本杀群、固定局头组织的多人包场
- 核心价值：将散乱的拼车沟通收束为结构化流程，降低局头组织成本，提升玩家出行体验

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 局头 | 微信授权/手机号 | 创建活动、招募车源、分组管理、发布变更通知 |
| 车主玩家 | 微信授权/手机号 | 浏览活动、响应车源、打卡签到 |
| 普通玩家 | 微信授权/手机号 | 浏览活动、报名参加、打卡签到 |

### 2.2 功能模块

1. **首页**：同城活动流、搜索筛选、快速发布入口
2. **活动详情页**：剧本信息、车源招募区、玩家列表、实时状态
3. **创建活动页**：剧本/店名/时间/人数 + 车源招募配置
4. **车源响应页**：车主填写出车信息（上车区域、可带人数、是否等散场）
5. **车队分组页**：局头按位置分组，生成去程/返程车队名单
6. **打卡通知页**：集合点打卡、临时变更通知、迟到/改坐提醒

## 3. 核心流程

**局头发布活动并招募车源**：局头创建包场活动 → 填写剧本信息和车源需求 → 活动发布到首页 → 车主看到活动并响应 → 局头收到车源通知 → 局头按位置分组 → 生成去程/返程车队名单 → 分享到群 → 玩家集合打卡 → 临时变更实时通知

    "一键分享至群聊" --> "活动日：集合打卡"

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深夜靛蓝 #1A1A2E + 霓虹洋红 #E94560，营造剧本杀悬疑氛围
- **辅助色**：暗紫 #16213E、暖金 #F0C27F 用于高亮和补贴标签
- **按钮风格**：圆角胶囊按钮，主要操作用洋红渐变填充，次要操作用半透明描边
- **字体**：标题使用 ZCOOL QingKe HuangYou（站酷庆科黄油体）营造趣味感，正文使用 Noto Sans SC
- **布局**：移动端底部导航，卡片式内容流，沉浸式深色主题
- **图标风格**：线性图标 + 微光晕效果，贴合悬疑夜行主题
- **整体氛围**：暗色系 + 霓虹点缀 = 剧本杀深夜出行的仪式感

| 页面名称 | 模块名称 | UI 元素 |

### 4.3 响应式设计

- 移动优先设计（用户明确要求移动 App）
- 底部导航栏适配移动端手势操作
- 卡片和按钮尺寸适配触摸交互（最小点击区域 44px）
- 横向滚动标签支持滑动手势
- 桌面端自适应为侧边栏 + 更宽卡片布局
